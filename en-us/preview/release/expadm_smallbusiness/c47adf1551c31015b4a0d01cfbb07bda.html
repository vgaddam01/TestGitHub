Transaction File Fields

The transaction file may contain the following fields. Depending on your configuration, you may not see all fields.

Field

Max Length

Description/Format

Batch Date

10

The date the batch was closed.

Format: MM/DD/YYYY

Employee ID

48

The ID of the employee who submitted the expense transaction. Employee ID often also serves as a vendor ID for AP systems or Payroll ID for Payroll integrations.

Employee Last Name

32

The last name of the employee who submitted the expense transaction.

Employee First Name

32

The first name of the employee who submitted the expense transaction.

Report ID

32

The report ID associated with the expense transaction.

Format: Alphanumeric string. Guaranteed to be unique. Often used as a voucher number when integrating to AP systems.

Report Currency Alpha Code

3

Reimbursement currency in ISO alpha format for the report.

Example: USD, GBP

Report Submit Date

10

The date/time the report was submitted.

Format: MM/DD/YYYY

Report Processing Payment Date

10

The date that all approvals were completed and the report was moved to the Processing Payment status.

Format: MM/DD/YYYY

Report Name

40

The name of the report as entered by the employee.

Report Entry Expense Type Name

64

The expense type of the expense transaction.

Report Entry Transaction Date

10

The date of the expense transaction based on when it was incurred and/or the credit card receipt date.

Format: MM/DD/YYYY

Report Entry is Personal Flag

1

Whether the Is Personal check box was selected for the expense transaction.

Format: Y/N

Report Entry Description

64

Expense description as entered by the employee.

Report Entry Vendor Name

64

The vendor name for the expense transaction.

Report Entry Vendor Description

64

The vendor description for the expense transaction.

Report Entry Payment Code Code

4

Payment Type code indicating how employee paid for purchase.

Report Entry Payment Code Name

80

The Payment Code name for the expense transaction.

Report Payment Reimbursement Type

1

Indicates whether journal entry reimbursed by Expense Pay. Either:

1 = Expense Pay

blank = not Expense Pay

This is for transactions reimbursed using Expense Pay service.

Billed Credit Card Account Number

4

The last four digits of the credit card account number used to make the purchase. 

Billed Credit Card Account Description

255

Description of the card account with the liability for the charge.

Journal Payer Payment Type Name

80

Payer payment type name. One of these: Company Employee Tax Authority Label for VAT records.

Example: UKVAT

Journal Payee Payment Type Name

64

Payee payment type. One of these: Company Employee

Journal Amount

23

The amount of the expense transaction. Value, as credit or debit, of the amount to be exchanged between the payer and payee for this expense account code. (not an absolute value)

Examples: Value of zero, credit, or debit, as the following: • 0 (Zero) "0" • + (Plus / Debit) "+50.00" • - (Minus / Credit) "-50.00"

Is Billable

1

Whether the Is Billable check box was selected for the expense transaction.

Format: Y/N

Journal Account Code

48

Contains the account code associated with the expense type.

Journal Debit or Credit

2

Whether the transaction is a debit or credit. Either: DR = Debit CR=Credit Format: DR/CR

Payment Demand Company Cash Account Code

48

 This field is reserved for future use with Expense Pay.

Payment Demand Company Liability Account Code

48

 This field is reserved for future use with Expense Pay.

Estimated Payment Date

10

The date the expense is estimated to be paid.

NOTE: This field is only populated when using Expense Pay.

Format: MM/DD/YYYY

Tax Authority Name

50

Name of the tax authority that applies to this expense.

Tax Authority Label

5

5 character code that appears on the expense entry pages and represents the alphanumeric code for the tax authority type.

Example: UKVAT

Report Entry Tax Transaction Amount

23

Calculated tax amount for this expense in the spend currency.

Report Entry Tax Posted Amount

23

Calculated tax amount for this expense entry in the reimbursement currency.

Report Entry Tax Source

4

Determines how this data was derived:

CARD= Provided from company card

USER = Entered by employee

SYST = Calculated by system

PROC = Entered by processor

Report Entry Tax Reclaim Transaction Amount

23

Calculated amount of tax eligible for reclaim in the spend currency.

Report Entry Tax Reclaim Posted Amount

23

Calculated amount of tax eligible for reclaim in the reimbursement currency.

Report Entry Tax Code

20

Tax code assigned to this tax authority for the expense type entered on the expense entry.

Report Entry Tax Reclaim Domestic Flag

1

Y = this tax configuration is defined as a reclaimable local tax jurisdiction. Format: Y/N

Report Entry Tax Adjusted Amount

23

Amount of tax based on approved amount, not total in the employees reimbursement currency.

Report Entry Tax Reclaim Adjusted Amount

23

Reclaim amount calculated on adjusted tax amount.

Report Entry Tax Allocation Reclaim Code

20

Reclaim code from the tax definition.

Report Entry Tax Reclaim Trans Adjusted Amount

23

Amount of tax based on approved amount, not total in spend transaction currency.

Report Entry Tax Allocation Reclaim Code

20

Allocation Reclaim Code.

Report Entry Total Tax Posted Amount

23

The sum of the tax calculated based on the expense amount.

Net Tax Amount

23

Calculated as Posted Amount minus Total Tax Posted Amount.

Report Entry Total Reclaim Adjusted Amount

23

The sum of the reclaimable tax amount calculated from the Total Tax Adjusted Amount.

Net Adjusted Reclaim Amount

23

Calculated as Approved Amount minus Total Reclaim Adjusted Amounts.

Custom Cost Tracking Fields

Varies

The custom cost tracking fields you configured in Setup will appear at the end of the file.
