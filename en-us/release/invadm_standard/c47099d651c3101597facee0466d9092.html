Voiding a payment request

If the financial system will not pay a payment request, or the request is deemed invalid, the Invoice Processor Manager role may change the request's Payment Status to Voided. This is useful where a request is rejected or cancelled in the client's financial system and the client wishes to apply this status to the request within Invoice as well. When a request is voided, a record of the request remains in the system, but the request is not included in any listing or extract of general requests. However, voided requests can be included in search result lists by filtering the search where Payment Status is equal to Voided.

The payment request can be voided in batches, at the Process Requests page level, or at the request level by opening the request directly in summary view. The procedure below explains how to do this at the Process Requests page level.

Note In order for a payment request to be voided, it must first be extracted.

To void a payment request:

  1. On the Process Requests page, select the payment request you want to void by clicking its check box.

  2. On the Actions menu, click Void Request. The audit log is updated.

  3. The request is now assigned a Payment Status of Voided.
